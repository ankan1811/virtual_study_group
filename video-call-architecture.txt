VIDEO CALL ARCHITECTURE — What's In-House vs What Agora Handles
================================================================


WHAT AGORA HANDLES (third-party service)
----------------------------------------

Agora provides the real-time media transport layer. Specifically:

1. WebRTC Infrastructure
   - TURN/STUN servers for NAT traversal (connecting users behind firewalls)
   - Media relay servers that route audio/video packets between peers
   - Adaptive bitrate encoding — adjusts quality based on network conditions
   - Packet loss recovery, jitter buffering, echo cancellation

2. Channel Management
   - Users join a named "channel" via client.join(appId, channelName, token, uid)
   - Agora's servers match users in the same channel and route media between them
   - Handles user-published / user-unpublished / user-joined / user-left events

3. Media Track Capture
   - createCameraVideoTrack() — captures camera via browser's getUserMedia API,
     wrapped with Agora's encoding/optimization pipeline
   - createMicrophoneAudioTrack() — same for microphone
   - Handles codec negotiation (VP8 in our case)

4. Publishing & Subscribing
   - client.publish(track) — sends local media to Agora's servers
   - client.subscribe(remoteUser, mediaType) — receives remote media from servers
   - Agora handles the SDP negotiation, ICE candidates, DTLS handshake internally

5. Device Change Detection
   - onCameraChanged() / onMicrophoneChanged() — notifies when devices are
     plugged in or removed

In short: everything that happens BETWEEN the two browsers (network transport,
encoding, relay, quality adaptation) is Agora.


WHAT'S BUILT IN-HOUSE (our code)
---------------------------------

Everything else — which is the majority of the user-facing application:

1. Room & Session System (backend)
   - backend/src/db/queries/roomSessions.ts
     Creates/manages room sessions with unique IDs (rm_<UUID>)
     Tracks session ownership, expiry (12-hour TTL)
   - backend/src/controllers/RoomSessionController.ts
     REST API: POST /room/session (create/resume), GET /room/sessions (list)
   - Each user has a permanent personal roomId (user_<userId>) assigned at login
   - Session roomIds are generated separately for time-bounded study sessions

2. Call Usage / Rate Limiting (backend)
   - backend/src/helpers/callUsage.ts
   - backend/src/controllers/CallUsageController.ts
   - Tracks per-user daily call seconds (1-hour daily limit)
   - GET /room/call-usage — returns remainingSeconds
   - POST /room/call-usage — reports deltaSeconds consumed
   - Uses Redis for fast per-user usage tracking
   - Frontend syncs usage periodically during call and on page unload (keepalive fetch)

3. Invite / Join Flow (frontend)
   - frontend/src/pages/JoinRoomPage.tsx
     Handles /join/:roomId links — validates auth, dispatches room entry to Redux,
     redirects to the call page with roomId in router state
   - Invite link is constructed as: ${origin}/join/${roomId}
   - Clipboard copy with fallback for older browsers
   - Socket-based invites also available (sendInvite event with roomId + inviterName)

4. Room State Management (frontend)
   - frontend/src/store/RoomStore/roomSlice.ts
     Redux slice tracking currentRoomId and isOwner
   - RoomId resolution priority: router state > Redux > user's personal roomId
   - Navbar "My Room" dispatches enterRoom with user.roomId

5. Call UI & Controls (frontend)
   - frontend/src/pages/RoomCallPage.tsx — the main orchestrator:
     * Pre-call lobby ("Room Ready" screen with Start Video Call button)
     * Agora client lifecycle (join, publish, subscribe, leave)
     * Track management (create, enable/disable, close, dispose)
     * Auto-publish video+audio sequentially when call starts
     * Multi-participant remote user tracking (remoteUsers[] array of RemoteUserInfo)
     * Dedup guard on user-joined to prevent ghost slabs on reconnect
     * Agora UID → app identity mapping via Socket.IO (agora:register / agora:uid-map)
     * uidToName state maps Agora UIDs to real user names (replaces old index-based chat name derivation)
     * Remote video tracks stored in state (videoTrack field on RemoteUserInfo)
       — Stream component plays them via ref when DOM mounts, eliminating the old
       fragile 50ms setTimeout hack
     * Call usage sync to backend during call
     * Browser tab close warning for unsaved chat messages
   - frontend/src/components/Stream.tsx — the video UI:
     * Dynamic grid layout: 1-col (solo), 2-col (2–4 people), 3-col (5+)
     * RemoteVideoTile component — plays remote video via useRef + useEffect
       (passes HTMLElement directly to track.play(), no string ID lookup)
     * Camera/mic toggle buttons (toggle local track, NOT Agora publish)
     * Publish button (pushes track to Agora for remote users to see)
     * End call button
     * Voice activity detection (custom useVoiceActivity hook using Web Audio API
       AnalyserNode — NOT Agora, pure browser API)
     * Speaking indicator (green ring + pulsing dot)
     * Remote user avatar with initials when camera is off
     * "Waiting..." placeholder when no remote user has joined

6. Real-Time Chat (runs alongside video)
   - frontend/src/components/ChatTabPanel.tsx + ChatComponent.tsx
   - Socket.IO based room chat (joinRoom event with name)
   - backend/src/socketServer.ts handles:
     * Room participant tracking (roomParticipants map)
     * Broadcasting participant lists (room:participants:${roomId})
     * Join/leave bot messages
     * Message persistence to database
     * Agora UID identity relay (agora:register → agora:uid-map:${roomId})
       Each client emits agora:register with their Agora UID after joining
       the channel; server broadcasts the UID→userId→userName mapping to
       the room so all clients can display correct names on video tiles

7. AI Features (alongside video)
   - frontend/src/components/AiPanel.tsx — AI doubt-solving tab
   - Chat summary generation via generateAndSaveSummary()
   - Whiteboard accessible from call page (/whiteboard/:roomId)

8. Navigation & Auth Integration
   - frontend/src/components/Navbar.tsx — "My Room" entry point
   - Auth-gated: JoinRoomPage redirects to login if unauthenticated,
     stores pending roomId in sessionStorage for post-login redirect
   - Exit handling: leave Agora channel, dispatch leaveRoom, navigate home


SUMMARY
-------

+------------------------------------------+---------------------------+
|            IN-HOUSE (our code)            |     AGORA (third-party)   |
+------------------------------------------+---------------------------+
| Room/session creation & management       | WebRTC transport layer    |
| User auth & invite links                 | TURN/STUN servers         |
| Call usage tracking & daily limits       | Media encoding (VP8)      |
| Redux state for room membership          | Adaptive bitrate          |
| Video call UI (dynamic grid, controls)   | Packet loss recovery      |
| Multi-participant layout (1/2/3-col)     | Echo cancellation         |
| Voice activity detection (Web Audio API) | Channel-based routing     |
| Camera/mic toggle logic                  | Publish/subscribe system  |
| Track lifecycle (create/dispose/rebind)  | Device change detection   |
| Auto-publish orchestration               |                           |
| Agora UID → identity mapping (Socket.IO) |                           |
| Remote user presence & name display      |                           |
| Real-time chat (Socket.IO)               |                           |
| AI panel & chat summaries                |                           |
| Whiteboard integration                   |                           |
| Participant list via sockets             |                           |
+------------------------------------------+---------------------------+

Roughly 80% of the video call feature is in-house code. Agora provides the
critical but narrow piece: getting raw audio/video bits from one browser to
another reliably. Everything the user sees, interacts with, and every business
rule (sessions, rate limits, invites, chat, AI) is our own code.
